# SmallDesktopDisplay 单按钮设置交互与后台刷新解耦优化设计

## 背景

上一轮已经把单按钮设置导航框架落地，设备现在具备：

- 首页短按提示气泡、长按进入设置
- 菜单页短按切换、长按执行
- 亮度调节页
- 运行状态检查页
- 重启确认页

但在真机使用中，出现了几类明显问题：

1. 系统每隔约 1 分钟会弹出 WiFi 连接进度条页面，打断当前页面。
2. 首页右下角动画在进入设置后仍然继续播放。
3. 运行状态检查页里的 WiFi 状态容易显示为 `not connected`，与用户认知不一致。
4. 内容页当前是“短按返回、长按返回”，不适合未来承载更多信息。
5. 长按过程中没有任何反馈，用户不知道什么时候已经达到长按触发阈值。

这轮工作不是新增一个孤立功能，而是要把“后台刷新、页面作用域、内容页语义、长按反馈”收敛成一套一致的交互与运行时模型。

## 目标

1. 后台天气刷新不再抢占当前页面，也不再复用开机联网的全屏进度 UI。
2. 右下角动画严格限定在首页显示，离开首页后立即停止。
3. 内容页统一改为：
   - 短按：向下滚动 / 移动焦点
   - 长按：返回上一层
4. 运行状态检查页升级为可滚动的信息页，补足网络相关上下文，避免误导。
5. 长按过程增加清晰、轻量、低成本的顶部进度反馈。
6. 保留当前已被接受的顶部标题与副标题布局，不在这一轮重新做页面整体视觉改版。

## 非目标

- 这轮不新增新的设置菜单项。
- 这轮不把设置系统扩展成完整的多级配置中心。
- 这轮不引入复杂动画转场或重做整套视觉语言。
- 这轮不重构天气获取、NTP、EEPROM 的业务逻辑，只处理它们对前台交互的呈现方式。
- 这轮不改按钮硬件、不改为多按钮方案。

## 方案比较

### 方案 A：交互与后台刷新一次性解耦

把后台联网模式、动画页面作用域、内容页滚动模型、长按过程反馈一起收口到统一模型。

- 优点：根因一次处理掉，后续新增页面和设置项可以直接复用。
- 缺点：会同时改 `Input`、`AppCore`、`AppDriver`、`Screen`、`Net`。

### 方案 B：局部修补

只隐藏后台刷新进度条，补停动画，微调诊断页文案。

- 优点：改动小。
- 缺点：内容页和长按反馈仍然不完整，后续还要再拆一次。

### 方案 C：先做纯 UI 反馈

先加长按反馈和内容页排版，后台联网行为维持现状。

- 优点：视觉体验能快速改善。
- 缺点：最打断人的问题仍然存在，不值得先做。

### 结论

采用方案 A。  
这轮定位为“中等范围重构”，目标是把前台交互与后台同步彻底分层，而不是继续累积症状修补。

## 根因分析

### 1. 后台刷新复用了开机联网路径

当前 `refreshDue` 触发后仍然走与启动阶段相同的 `ConnectWifi` 路径。底层联网实现没有区分“前台阻塞联网”和“后台静默联网”，因此后台同步也会绘制进度页面。

### 2. 诊断页只看实时链路状态

当前运行状态检查页读取的是抓快照时的 `WiFi.status() == WL_CONNECTED`。  
而固件在后台同步完成后会主动休眠 WiFi，因此“现在没连上”并不等于“系统网络异常”，只是当前无线链路处于省电状态。

### 3. 动画刷新是全局 tick

主循环当前无条件调用动画 tick，动画层也没有“当前是否首页”的概念，所以只要动画功能启用，就会继续向屏幕右下角绘制。

### 4. 内容页还是静态一屏思维

当前信息页模型固定为少量静态行，没有焦点、滚动窗口或分页状态，因此交互只能退化成“按一下就返回”。

### 5. 输入层只有终态事件

当前输入层只上报 `ShortPress` 和 `LongPress` 两个结果，没有“按下中”“长按已就绪”“释放”这类中间状态，UI 没法在长按过程中给用户反馈。

## 设计原则

- 前台页面永远优先于后台同步提示。
- 后台任务可以失败，但不能打断当前页面。
- 页面作用域必须明确：首页元素只属于首页。
- 内容页交互必须能支撑“超过一屏的信息”。
- 长按反馈要轻量、确定、低资源占用，优先可读性而不是炫技。
- 保持当前顶部标题区布局，不为视觉微调引入额外结构复杂度。

## 设计方案

### 1. 引入联网呈现模式：前台阻塞 vs 后台静默

联网动作必须区分两种模式：

- `ForegroundBlocking`
  - 用于开机首次联网
  - 允许显示全屏进度
  - 若凭据无效，允许进入配网流程
- `BackgroundSilent`
  - 用于定时天气刷新
  - 不允许显示全屏进度
  - 不允许突然拉起配网页面
  - 若失败，只记录后台刷新失败并保留当前页面

这意味着底层联网接口不能再只有一个无上下文的 `connect()`。  
它必须知道自己是“开机联网”还是“后台静默重连”。

### 2. 后台刷新期间的前台行为

后台刷新开始时：

- 当前页面保持不变
- 当前输入语义保持不变
- 设置页、内容页、确认页都不应被覆盖

后台刷新成功时：

- 首页数据可在下一次 render 时更新
- 非首页页面不强制跳回首页

后台刷新失败时：

- 不显示阻塞错误页
- 不弹出配网逻辑
- 只记录失败状态，供首页小提示或诊断页查看

### 3. 动画严格绑定首页作用域

动画层必须增加“首页是否活跃”的约束。

规则：

- 只有当当前 `view` 是首页时，动画 tick 才允许绘制
- 进入设置页、内容页、调节页、确认页时，动画立即暂停
- 返回首页后再恢复

为了避免页面切换后残留动画像素，离开首页时应清掉动画区域一次，之后不再刷新。

### 4. 内容页统一升级为可滚动信息页

所有内容页统一采用：

- 短按：向下移动当前焦点
- 若焦点移出当前可视窗口，则窗口向下滚动一行
- 到达尾部后，再次短按回到顶部
- 长按：返回上一层

这套规则适用于：

- 纯信息页
- 将来有更多行的状态页
- 需要阅读多屏内容的说明页

即使一页内容暂时没有超过一屏，也仍然保留焦点移动，这样用户能感知“短按是在浏览内容，不是在返回”。

### 5. 运行状态检查页改为“可解释的状态页”

诊断页仍然坚持“进入时抓一次快照，不自动刷新”，但展示内容要从“裸字段”改成“用户能理解系统为何这样工作”的信息页。

建议至少展示以下条目：

1. `Saved SSID`
   - 已保存的 WiFi 名称
2. `WiFi Radio`
   - `awake` / `sleeping`
3. `WiFi Link`
   - `connected` / `disconnected`
4. `Active SSID`
   - 当前链路已连接时显示真实 SSID
   - 未连接时显示 `-`
5. `Last Weather Sync`
   - 最近一次成功天气同步时间
6. `Refresh Interval`
   - 当前天气刷新周期（分钟）
7. `Free Heap`
8. `Flash Used`
9. `Flash Total`

这样即使 WiFi 当前处于休眠状态，用户也能看见：

- 系统保存了哪个 SSID
- 当前是不是在省电休眠
- 最近一次成功同步是什么时候

不会再把“当前没连着 AP”误解成“设备坏了”。

### 6. 长按反馈采用顶部左到右进度线

本轮不采用中心放射线，而采用顶部细进度线。

原因：

- 更节省绘制成本
- 更容易与现有顶部标题区融合
- 在 240x320 TFT 上更清楚
- 不会遮挡正文区

行为定义：

- 按键稳定按下后，顶部出现一条细线
- 细线从左到右增长
- 达到长按阈值时，线条满格并闪一下
- 满格后保持一个“已就绪”状态，告诉用户此时松手即可触发长按语义
- 若提前释放，则线条消失，按短按处理

### 7. 输入层新增“长按就绪”中间状态

为了支持上述反馈，输入层需要新增中间状态，而不仅仅是最终结果事件。

建议输入层提供以下离散事件：

- `PressStarted`
- `LongPressArmed`
- `PressReleased`
- `ShortPress`
- `LongPress`

其中：

- `ShortPress` / `LongPress` 继续负责真正的页面行为
- `PressStarted` / `LongPressArmed` / `PressReleased` 只负责 UI 反馈

当前使用的 Button2 已经具备：

- `setPressedHandler`
- `setReleasedHandler`
- `setLongClickDetectedHandler`

因此这套反馈不需要更换输入库，也不需要手写底层 GPIO 扫描。

### 8. 保持当前顶部布局，不做视觉回退

当前顶部标题较高、副标题留在上方的布局，用户已接受。  
这一轮不调整：

- 标题高度
- 顶部副标题位置
- 页面整体视觉基调

本轮只修复行为与信息表达，不重做页面构图。

## 数据模型建议

### `DiagnosticsSnapshot`

扩展为能承载“当前链路 + 当前省电状态 + app 层上下文”的条目。

建议包含：

- `freeHeapBytes`
- `programFlashUsedBytes`
- `programFlashTotalBytes`
- `savedWifiSsid`
- `activeWifiSsid`
- `wifiLinkConnected`
- `wifiRadioAwake`
- `lastWeatherSyncEpoch`
- `refreshIntervalMinutes`

### `InfoPageState`

新增统一信息页滚动状态：

- `selectedRowIndex`
- `firstVisibleRowIndex`

### `HoldFeedbackState`

新增长按反馈状态：

- `visible`
- `armed`
- `pressStartedMs`
- `progressPercent`

## 事件与动作建议

### `AppEvent`

在现有事件之外新增：

- `pressStarted(nowMs)`
- `longPressArmed(nowMs)`
- `pressReleased(nowMs)`

说明：

- `shortPressed` / `longPressed` 继续保留，用于真正执行交互
- 新增事件只用于顶部长按反馈状态机

### `AppAction`

需要把联网动作扩展为带上下文的请求，而不是只有一个裸 `ConnectWifi`。

本轮明确采用：

- `ConnectWifi` + `payload.mode`

其中 `mode` 至少区分：

- `ForegroundBlocking`
- `BackgroundSilent`

选择这一种方式的原因是：

- 不会让动作枚举快速膨胀
- 后续若引入“用户主动联网检查”“用户主动重试同步”等模式时，仍能沿用同一动作类型
- `AppDriver` 与底层 `NetworkPort` 可以共享同一套联网入口，只在呈现策略上分叉

## 页面规则总结

### 首页

- 短按：显示调试气泡
- 长按：进入设置
- 右下角动画：允许播放

### 菜单页

- 短按：选中下一项
- 长按：执行当前项
- 右下角动画：禁止播放

### 内容页

- 短按：向下移动焦点 / 向下滚动
- 长按：返回
- 右下角动画：禁止播放

### 调节页

- 短按：切到下一档并预览
- 长按：保存并返回
- 右下角动画：禁止播放

### 后台刷新

- 允许发生在任意页面
- 不允许抢占页面
- 不允许显示全屏联网进度
- 不允许自动进入配网流程

## 测试要求

### Host 侧

至少补以下测试：

- 后台刷新触发时，不改变当前 `UiRoute`
- 后台刷新失败时，不进入阻塞错误页
- 内容页短按移动焦点并在需要时滚动窗口
- 内容页长按返回上一层
- 首页之外动画标志为禁用
- 长按反馈状态：
  - `PressStarted` 后可见
  - `LongPressArmed` 后进入已就绪
  - `PressReleased` 后清空

### 嵌入式编译验证

- `esp12e` 编译通过
- 后台刷新时不再出现全屏进度条
- 进入设置页后右下角动画停止
- 诊断页显示保存的 SSID、链路状态、省电状态和最近同步时间
- 长按时顶部进度线正常显示并在就绪后给出明确反馈

## 风险与权衡

### 风险 1：输入事件变多后状态机复杂度上升

控制方式：

- 保持“反馈事件”和“行为事件”职责分离
- 不让 `PressStarted` 直接触发页面跳转

### 风险 2：后台静默联网与首次联网路径分叉

控制方式：

- 只在联网呈现模式层分叉
- 不复制天气、NTP、存储等业务逻辑

### 风险 3：信息页滚动引入新的渲染复杂度

控制方式：

- 抽成统一 `InfoPage` 滚动渲染
- 诊断页先作为第一个使用者

## 结论

这轮优化的核心不是“再加几个小补丁”，而是把以下四个边界彻底立住：

- 后台刷新不能抢前台
- 首页元素只能存在于首页
- 内容页必须支持滚动语义
- 长按必须有过程反馈

只要这四条边界立住，后续无论再加 WiFi 设置、刷新周期、方向设置、关于页，都会落在同一套稳定框架上。
